# 基于 LangGraph 的多 Agent 网文生成与审稿系统

这是一个面向长篇网文创作的 AI 多 Agent 工作流项目。

它不是简单地让一个大模型直接写小说，而是把长篇小说创作拆成多个协作环节：世界观生成、角色卡生成、篇章规划、章节拆分、正文生成、审稿、修订、剧情圣经维护、反重复检测和 TXT 反向建档。

项目适合用于学习和演示：

- LangGraph 多 Agent 工作流编排
- FastAPI AI 应用后端开发
- 长篇文本生成中的长期记忆设计
- AI 审稿与修订闭环
- 小说大纲、剧情圣经、章节事件账本等结构化创作系统

## 功能特性

### 小说创建

- 根据一句创意生成小说基础设定
- 自动生成世界观
- 自动生成角色卡
- 自动生成篇章大纲
- 自动生成节奏段落
- 自动生成章节大纲

### 三层大纲体系

系统将长篇网文结构拆成三层：

- `volume_outline`：篇章大纲，规划 30-50 章的阶段性主线
- `pacing_outline`：节奏段落，控制开局、升级、反转、高潮等节奏
- `outline`：章节大纲，记录每章标题、梗概、冲突、关键场景和钩子

### 多 Agent 创作流程

项目中的主要 Agent 包括：

- 创意架构师：负责世界观、角色卡、大纲生成
- 分章编辑：负责扩展章节大纲
- 网文作家：负责生成章节正文
- 审稿编辑：负责检查章节质量和重复问题
- 连续性编辑：负责检查设定、人设、大纲一致性
- 修订作家：负责根据问题重写章节
- 剧情档案员：负责更新剧情圣经和章节事件账本
- 大纲修订编辑：负责根据审稿报告更新大纲
- 导入档案员：负责从 TXT 小说反向重建项目结构

### 章节正文生成

- 根据章节大纲生成正文
- 自动参考上一章摘要
- 自动参考最近章节事件
- 避免重复旧桥段
- 保存前检查正文是否贴合当前章节大纲

### AI 辅助更新

如果对已有设定不满意，可以输入修改方向，让 AI 辅助更新：

- 世界观
- 角色卡
- 篇章大纲
- 节奏段落
- 章节大纲
- 剧情圣经

同时支持新增角色卡、新增篇章大纲、新增章节大纲。

### 审稿与修订

系统支持整本审稿和章节检查，能够发现：

- 世界观冲突
- 角色人设冲突
- 章节正文偏离大纲
- 剧情圣经冲突
- 章节重复
- 桥段重复
- 节奏错位
- 伏笔遗漏

审稿后可以根据审稿报告更新大纲，并同步重写受影响章节。

### 反重复机制

为了减少 AI 长篇写作中常见的“同一件事换个说法又写一遍”，系统加入了多层反重复机制：

- 最近章节文本相似度检测
- 章节事件账本
- 重复黑名单
- 相似历史事件检索
- 大纲关键词命中检查
- 换皮重复检测

### 剧情圣经

剧情圣经用于维护长篇小说的长期记忆，包括：

- 时间线
- 角色状态
- 未回收伏笔
- 已回收伏笔
- 地点和物件状态
- 连续性备注
- 上一章摘要
- 章节事件账本
- 重复黑名单

### TXT 导入与反向建档

支持导入已有 TXT 小说，并自动：

- 拆分章节
- 根据正文生成章节大纲
- 补全世界观
- 补全角色卡
- 补全篇章大纲
- 补全剧情圣经

适合把已有小说迁移回系统继续编辑。

### 小说回收站

删除小说时不会直接永久删除，而是移动到回收站目录，支持：

- 恢复误删小说
- 永久删除回收站小说

## 安装

建议使用 Python 3.10 或更高版本。

```bash
git clone https://github.com/your-name/your-repo.git
cd your-repo
pip install -r requirements.txt
```

## 环境变量

请创建 `.env` 文件，或者在系统环境变量中配置：

```env
OPENAI_API_KEY=
NOVEL_LLM_MODEL=gpt-4o-mini
OPENAI_API_BASE=
OPENAI_BASE_URL=
NOVEL_MOCK=0
```

说明：

- `OPENAI_API_KEY`：你的模型 API Key
- `NOVEL_LLM_MODEL`：模型名称，默认可使用 `gpt-4o-mini`
- `OPENAI_API_BASE` / `OPENAI_BASE_URL`：OpenAI 兼容接口地址，可用于第三方模型服务
- `NOVEL_MOCK=1`：开启模拟模式，不调用真实模型

请不要把真实 `.env` 文件提交到 GitHub。

## 启动

```bash

```

```text

```
